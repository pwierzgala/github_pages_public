Math¶

To enable support for LaTeX syntax add one of the following to the sphinx extensions:

  • sphinx.ext.mathjax

  • sphinx.ext.imgmath

References¶

Named references¶

Named references end with a single underscore. They should have unique names, otherwise during a documentation build the following warning will be displayed:

*WARNING: Duplicate explicit target name: <name>*

Google

`Google <http://google.com>`_

Anonymous references¶

Anonymous references end with a double underscore.

Google

`Google <http://google.com>`__

References to headers within a document¶

Sphinx automatically creates header labels which available within a document as lowercased and hyphenated headers.

References to headers

`References to headers <#references-to-headers>`__

References to any place in a documentation¶

In order to create a reference to a place in a documentation a label in that place has to be created. Then it’s possible to refer the label with ref directive. For example, a some_label label was defined below.

  • Reference to some label

.. _some_label:

-   Reference to :ref:`some label <some_label>`

Headers¶

Suggested symbols for marking headers:

  1. # + two blank lines before.

  2. * + two blank lines before.

  3. =

  4. -

  5. ^

  6. "

First and second level headers should be preceded by two blank lines.

##################
H1: document title
##################

Content.


**********
H2: header
**********

Content.


******************
H2: another header
******************

Content.

H3: header
==========

Content.

H4: header
---------

Content.

H5: header
^^^^^^^^^^

Content.

H6: header
""""""""""

Content.
